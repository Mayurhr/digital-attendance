Campus GPS Settings

Defines the campus boundary used when a teacher enables GPS restriction on an attendance session. This validates the device running the session is on campus — see the note below for what it does and doesn't verify.

What this does and doesn't verify

This checks that the device running the attendance session (the teacher's laptop/phone) is within the radius above when a session starts. It's a real deterrent against attendance being run entirely from off-campus. It cannot verify each individual student's location — recognition happens through one shared camera/scanner, not one device per student, so per-student location checking isn't something this architecture can do. It also doesn't catch a card scanned on someone else's behalf, or a face misrecognition — those are accuracy problems, not location problems.
